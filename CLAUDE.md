# CLAUDE.md

## このリポジトリの目的

エンジニア（リポジトリ所有者本人）の **スキルシート（職務経歴書）** を作成・管理するためのリポジトリ。
成果物は「人に提出する経歴書」そのものであり、アプリやツールのコードではない。

- 主な読み手：取引先・採用担当・営業担当・エージェント
- 想定フォーマット：Markdown を原本とし、必要に応じて PDF / Word に書き出す
- 言語：日本語（固有名詞・技術名は原表記のまま）

## 作業方針

スキルシートは「事実の記録」であり、誇張・創作は信用を損なう。以下を厳守する。

- **事実を捏造しない。** 経験していない技術・期間・役割・成果を勝手に追加しない。
  情報が足りない箇所は埋めずに、本人に確認する形で質問する。
- **数値・固有名詞は推測で補わない。** 期間、人数、規模、改善率などは本人提供の値だけを使う。
  不明なら「（要確認）」と明示する。
- **守秘義務に配慮する。** 客先名・プロジェクト名は伏せる前提で書く（後述の「載せない情報」参照）。
- **表記を統一する。** 技術名（例：React, TypeScript, AWS）、日付形式（YYYY/MM）、
  箇条書きスタイルを一貫させる。既存記述のスタイルに合わせる。
- **トーンは簡潔・客観。** 「〜を担当」「〜を設計・実装」のように動詞で実績を示す。

## 編集時の注意

- 既存ファイルのフォーマットを最優先で踏襲する。新しい章立てや書式を勝手に導入しない。
- 内容を「盛る」提案はしてよいが、本人の確認なしに本文へ反映しない。
- 個人情報（氏名・連絡先・住所など）の扱いは本人の指示に従う。

## 構成

```
.
├── CLAUDE.md                  … この設定ファイル
├── profile.sample.md          … 本人情報マスターの雛形（公開）
├── profile.json               … 構造化マスター（profile.md の生成元・.gitignore）
├── profile.md                 … profile.json から自動生成（直接編集しない・.gitignore）
├── samples/                   … エージェント/会社のフォーマット見本（公開）
├── import/                    … 取り込み元の既存スキルシート（.gitignore）
├── output/                    … 生成した提出用スキルシート（.gitignore）
├── tools/
│   └── profile-editor/        … profile.json を編集する GUI（Python 標準ライブラリのみ）
└── .claude/skills/
    ├── generate-skillsheet/   … profile + sample → 提出用に整形して出力するスキル
    ├── import-profile/        … import/ の既存スキルシート → profile.md に転記するスキル
    ├── refine-profile/        … profile.md を点検し、不足を本人に質問して埋めるスキル
    └── calc-experience/       … 経歴の期間×技術から各技術の経験年数を現在日付で計算・更新するスキル
```

> マスターは **`profile.json`**。`profile.md` はそこから自動生成される（手で編集しない）。
> 編集は GUI（`python3 tools/profile-editor/server.py`）または `profile.json` を直接。
> スキルが `profile.md` を更新した場合は、内容を `profile.json` にも反映して整合を保つこと。

### 公開ポリシー（public リポジトリ想定）

- **公開する**：雛形（`*.sample.md` / `samples/`）、スキル、`CLAUDE.md`。
- **コミットしない**（.gitignore 済み）：実データ `profile.md`、取り込み元 `import/`、生成物 `output/`。
- 個人情報は `profile.md` / `import/` / `output/` の中だけに置く。公開対象ファイルには書かない。

### データの流れ

1. `profile.sample.md` を **`profile.md`** にコピーして実データを記入（マスター）。
   既存のスキルシートがあれば `import/` に置いて **`import-profile`** で取り込める。
2. 提出先のフォーマット見本を **`samples/`** に入れる。
3. **`generate-skillsheet`** スキルが profile を見本に合わせて整形し、**`output/`** に出力する。

> 原則：`profile.md` が単一の真実。出力は使い捨てで、`samples/` の見本ごとに作り直す。
