# refine-profile

`profile.md`（事実のマスター）を点検し、不足・曖昧・空欄の箇所を **本人への質問** で
埋めていくスキル。提出物（output/）は作らない。事実を増やす／精度を上げることが目的。

## 手順

1. **profile.md を読む**
   - 各セクション（基本情報 / スキルサマリ / 技術スキル / 担当可能工程 /
     プロジェクト経歴 / 資格 / 公開実績 / 自己PR）の現状を把握する。

2. **不足・要確認を洗い出す**
   - 次のような箇所をリストアップする。
     - 空欄、または「（要確認）」が付いている項目
     - プロジェクト経歴で抜けている要素（期間・業種・役割・チーム規模・担当工程・
       使用技術・実績のいずれかが欠けている案件）
     - 数値で語れる実績が無い／曖昧な箇所（規模・件数・改善・期間など）
     - 技術スキルの経験年数・習熟度が空欄のもの
     - 終了日が未来日付など、矛盾・要確認の値

3. **質問する（埋める前に必ず聞く）**
   - 洗い出した不足を **質問形式** で本人に提示する。
   - 1回で大量に聞かず、関連するものを **数問ずつのまとまり** にして進める。
   - 優先順位：①矛盾・要確認の解消 → ②経歴の必須要素の欠落 → ③数値・実績の具体化 →
     ④スキルの経験年数/習熟度 → ⑤任意項目（自己PR・公開実績など）。
   - 推測の選択肢を示してもよいが、**本人が選んだ／答えた値だけ**を採用する。

4. **回答を profile.md に反映する**
   - 本人が答えた事実だけを該当セクションに記入する。
   - 既存の値と衝突する場合は上書き前に確認する。
   - 答えが得られなかった項目は空欄／「（要確認）」のまま残す。
   - 反映後、まだ残っている不足があれば次のまとまりを質問する（3〜4を繰り返す）。

## 厳守事項

- **捏造・誘導をしない。** 本人が言っていない技術・期間・役割・成果・数値を書かない。
  「たぶんこうですよね？」で勝手に確定させない。答えが無ければ空欄のまま。
- **数値は本人提供の値のみ。** 規模・人数・改善率・期間を盛らない。
- **会社業務と個人開発を混同しない。** 個人開発・OSS・学習は「プロジェクト経歴（会社業務）」に
  入れず、「自己PR」「公開実績」へ。
- **守秘義務。** 客先名・固有のプロジェクト名は profile.md にも書かない（業種・概要で表現）。
- **既存フォーマットを踏襲する。** profile.md の章立て・箇条書きスタイル・表記
  （技術名・日付 YYYY/MM）を崩さない。新しい章を勝手に足さない。
- **出力先は profile.md のみ。** スキルシート本体（output/）は生成しない。

## 反映後

- 「今回埋めた項目」「本人の回答待ちで残した項目」「まだ不足している箇所」を箇条書きで報告する。
- 次に聞くべき質問が残っていれば提示する。
