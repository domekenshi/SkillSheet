# generate-skillsheet

`profile.md`（事実のマスター）を、`samples/` にあるフォーマット見本に合わせて
整形し、提出用ファイルとして `output/` に出力するスキル。

## 手順

1. **対象フォーマットを決める**
   - ユーザーがフォーマットを指定していればそれを使う。
   - 指定がなく `samples/` に複数ある場合は、どの見本に合わせるか確認する。
   - `samples/` が空なら、汎用の職務経歴書フォーマットで出力してよいか確認する。

2. **見本を読む**
   - 指定された見本の **項目・章立て・順序・表記ルール・粒度** を把握する。
   - Excel/Word の場合は構造（列・セクション）を読み取る。

3. **profile.md を読む**
   - 出力に必要な事実だけを取り出す。

4. **当てはめて出力する**
   - 見本の構造に profile.md の事実をマッピングして `output/` に生成する。
     ファイル名は提出先がわかる名前にする（例：`output/agent-a_skillsheet.md`）。
   - 見本のフォーマット（見出し・表・順序）を最優先で踏襲する。

## 厳守事項

- **捏造しない。** profile.md にない技術・期間・役割・成果を足さない。
  見本にあって profile.md にない項目は、空欄か「（要確認）」にして
  本人に確認する。推測で埋めない。
- **数値・固有名詞を盛らない。** 期間・人数・規模・改善率は profile.md の値のみ。
- **守秘義務。** 客先名・プロジェクト名は伏せ、業種・概要で表現する。
- **表記統一。** 技術名・日付形式（YYYY/MM）・箇条書きスタイルを見本に合わせて一貫させる。
- **profile.md は書き換えない。** このスキルは出力のみ。事実の更新は別途本人の指示で行う。

## 出力後

- 「見本のどの項目を埋められなかったか（要確認の箇所）」を箇条書きで報告する。
- 客先名など伏せた箇所があれば明示する。
