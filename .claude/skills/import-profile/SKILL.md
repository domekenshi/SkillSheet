# import-profile

既存のスキルシート/職務経歴書を読み取り、その内容を **`profile.md`（マスター）** に
転記・追記するスキル。`generate-skillsheet` とは逆方向（取り込み）の役割。

## 手順

1. **入力元を特定する**
   - 既存スキルシートは原則 `import/` フォルダに置かれている（個人情報を含むため .gitignore 済み）。
   - ファイル（Markdown / Excel / Word / PDF など）か、貼り付けテキストか確認する。
   - `import/` に複数ある/場所が不明な場合は対象を確認する。

2. **profile.md の現状を読む**
   - 既に埋まっている項目と空欄を把握する。

3. **既存スキルシートから事実を抽出する**
   - profile.md の項目（基本情報 / 技術スキル / 担当工程 / 経歴 / 資格 / 公開実績 等）に
     対応する情報を読み取る。
   - 経歴は各案件ごとに 期間・業種・役割・チーム規模・担当工程・使用技術・実績 に分解する。

4. **profile.md に反映する**
   - 空欄には抽出した事実を埋める。
   - 既存の値と **食い違う/衝突する** 場合は、上書きせずに本人へ提示して確認する。
   - 元資料に無い項目は埋めず、空欄または「（要確認）」のままにする。

## 厳守事項

- **捏造・推測で補完しない。** 元資料に書かれている事実だけを転記する。
  習熟度・経験年数・数値が元資料に無ければ勝手に付けない。
- **勝手に上書きしない。** 既存の profile.md の値を消す/変える前に必ず確認する（追記が基本）。
- **正規化する。** 日付は YYYY/MM、技術名は表記統一、客先名は伏せて業種表現に直す。
  ただし元の事実そのもの（期間・役割・実績）は変えない。
- **出力先は profile.md のみ。** スキルシート本体（output/）は生成しない。

## 反映後

- 「どの項目を埋めたか」「埋められず（要確認）にした項目」「衝突して確認が必要な項目」を
  箇条書きで報告する。
