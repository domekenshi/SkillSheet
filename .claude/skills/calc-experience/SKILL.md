# calc-experience

`profile.md` の **プロジェクト経歴（会社業務）＋自己PRの個人開発** の「期間 × 使用技術」を
集計し、各技術の **経験年数を現在日付基準で計算** して技術スキル表に反映するスキル。
時間が経つと継続中案件の年数が増えるため、提出前のリフレッシュに使う。

## 基準日

- 「現在日付」を基準にする。ハーネスが渡す **currentDate** を使う。
  渡されていなければ本人に基準日を確認する（推測しない）。

## 手順

1. **profile.md を読む**
   - プロジェクト経歴の各案件から「期間」と「使用技術」を取り出す。
   - 自己PRの個人開発など、期間と技術が明記された活動も対象に含める。
     ただし会社業務と個人開発は内訳が分かるよう区別して扱う（後述の報告で明示）。

2. **期間を正規化する**
   - 各案件の期間を `開始(YYYY/MM) 〜 終了(YYYY/MM)` の月単位に変換する。
   - 「現在」「運用中」「〜継続中」など終了が未確定のものは **終了＝現在日付の月** とする。
   - 終了が未来日付（退職予定日など）の場合も、**算入は現在日付の月まで**にする
     （まだ経過していない期間を先取りしない）。

3. **案件ごとに「メイン技術 / サブ技術」を切り分ける**
   - 1案件で多数の技術を使っていても、**その案件で主に使った技術＝メイン**だけを
     年数に算入する。短期間だけ触ったもの＝サブは年数に効かせない
     （例：FE中心の案件で2カ月だけ触った Swift はサブ）。
   - メイン/サブの分類は **本人に確認する**（役割記述などから提案してよいが確定は本人）。
     確定した分類は profile.md のプロジェクト経歴に「メイン技術：…」として残すと再現可能。

4. **技術ごとに期間を集計する**
   - 技術名は表記ゆれを名寄せする（例：`React`/`React 19`、`Java 7`/`Java 8`→`Java`、
     `Vue2`/`Vue3`→`Vue.js`）。名寄せ規則は判断に迷えば本人に確認する。
   - **メインに設定した案件の期間だけ**を、月単位で **和集合**（重複除外・二重計上しない、
     現在日付cap）して合算する。サブ利用は加えない。
   - 合算した月数を `N年Mカ月` に変換する（12カ月＝1年、端数は切り捨て）。

5. **技術スキル表を更新する（加算方式）**
   - **採用値 = max(元値〔本人申告〕, メイン和集合)** とする。
     - 元値は下限として尊重し、勝手に減らさない（過去のサブ利用分なども元値に含まれるため）。
     - 継続中案件でメインの技術は現在日付まで伸びるので、再実行のたびに加算されていく。
     - どの案件でもサブの技術は和集合が空 → 元値を維持。
   - **既存値と食い違う場合は上書き前に提示して確認する。**
     特に元資料（import由来）の値は本人申告のため勝手に潰さない。
   - 計算根拠（どの案件をメインとして合算したか）を報告に添える。

## 厳守事項

- **推測で期間・技術を足さない。** profile.md に書かれた期間・使用技術だけを根拠にする。
  期間や技術が書かれていない案件は計算対象外（その旨を報告する）。
- **未経過期間を先取りしない。** 退職予定日など未来の終了日は現在日付までで止める。
- **「利用のみ」の環境は経験年数を付けない。** 自分で構築・運用しておらず、構築済み環境を
  利用しただけの技術（例：AWS などのクラウド/インフラ）は、経験年数の概念に馴染まないため
  年数を算出せず、備考に「利用のみ」と明記する。判断に迷えば本人に確認する。
- **重複期間を二重計上しない。** 並行案件は和集合で数える。
- **習熟度は触らない。** このスキルは「経験年数」列のみを対象にする。
- **基準日を明示する。** 計算に使った現在日付を報告に書く（後日見て再現できるように）。
- **出力先は profile.md のみ。** スキルシート本体（output/）は生成しない。

## 反映後

- 基準日、各技術の新しい経験年数、計算根拠（合算した案件と期間）を箇条書きで報告する。
- 既存値と差異がある技術、名寄せで迷った技術、計算対象外にした技術を明示する。
